1.3 3대 차별화 포인트
기존 글로벌 육아 앱에 모두 없는 3가지 핵심 기능으로 시장 빈틈을 공략한다.
① 분유/기저귀 자동 소진 알림
사용자가 이미 입력하는 수유량과 기저귀 사용 데이터에 구매 정보만 추가 입력하면, 일평균 소비량을 자동 계산해 &quot;3일 후 떨어짐&quot; 같은 사전 알림을 제공한다. 어필리에이트 매출과 직결되는 핵심 기능.
② 단골 병원 등록 + 예방접종 자동 매칭
자녀 생일과 단골 병원만 등록하면 국가별 표준 예방접종 일정에 맞춰 자동 알림. 원클릭 전화 연결과 길찾기 지원으로 부모의 실행 부담 최소화.
③ 부부 실시간 공유 + 광고 없음
Supabase Realtime 기반 실시간 동기화. 모든 핵심 기능이 무료. 광고 없는 깔끔한 UX 유지. 둘째 자녀부터만 유료.

2.3 핵심 시장 통찰
한국
저출생에도 불구하고 골드키즈 트렌드(텐 포켓)로 1인당 육아 지출은 오히려 증가. 부모들의 71.2%가 온라인 커뮤니티를 통해 육아 정보를 얻으며 디지털 의존도가 매우 높다. 키즈테크 시장이 빠르게 성장 중.

영어권
북미가 글로벌 시장의 36~45%를 차지하는 최대 시장. 75% 이상의 밀레니얼 부모가 디지털 플랫폼에 의존. Huckleberry, BabyCenter, Pebbi 등 주요 플레이어 다수. 프라이버시 민감도가 매우 높다.

4. 핵심 차별화 기능
4.1 분유 재고 자동 알림 시스템
기본 로직
사용자가 이미 분유량을 기록하고 있으니, 거기에 재고 정보만 더하면 자동 계산이 가능하다.
입력: 제품명, 용량(예: 800g), 구매일자, 가격, 구매처
자동 계산: 일평균 소비량 (최근 7일 데이터 기반)
잔량 = 구매량 - 누적 소비량
소진 예상일 = 잔량 ÷ 일평균 소비량
알림 트리거: 3일 전, 1일 전, 당일
화면 흐름
분유 등록 화면 — 제품 선택, 용량 입력, 개봉일
수유 기록 시 자동 차감 — 120ml 수유 시 약 17.6g 자동 차감
홈 위젯 — &quot;남은 분유: 약 4일 분량&quot;
푸시 알림 — &quot;분유가 3일 후 소진될 예정. 미리 주문하세요&quot;
원클릭 재구매 — 쿠팡/Amazon 어필리에이트 링크
기술적 디테일
분유 1g당 약 6~7ml 수유분 (제품마다 다름) → 제품 DB에 환산 계수 저장
개봉 후 유통기한(보통 1개월) 추적 → 별도 알림
여러 통 동시 보유 → FIFO(먼저 개봉한 것부터 차감) 로직
초기 보정 기간(7일 이내)은 &quot;예측 정확도 낮음&quot; 표시
4.2 기저귀 재고 + 사이즈 업 예측
입력: 브랜드, 사이즈(NB/S/M/L/XL), 매수, 구매일
자동 차감: 기저귀 교환 기록 기반
일평균 사용 매수 (신생아 8~12매, 6개월 6~8매, 돌 4~6매)
사이즈 업 예측 (보너스)
아기 체중 데이터 + 현재 사이즈로 &quot;곧 사이즈 업 필요&quot; 알림. 예: &quot;현재 8kg, M사이즈는 6~10kg 권장. 다음은 L사이즈 구매를 추천&quot;
4.3 단골 병원 등록 + 자동 알림
이상적 사용자 흐름
아기 생일 입력 (한 번만)
단골 소아과 등록 (한 번만)
앱이 자동으로: &quot;다음 주 화요일이 폐구균 3차 접종 시기예요&quot;
원클릭: [전화하기] [길찾기]

8. 기술 스택 및 구현 가이드
8.1 핵심 결정
Flutter (클라이언트) + Supabase (백엔드). 6개국 출시·빠른 개발·낮은 운영비를 모두 만족하는 강력한 조합.
8.2 왜 이 조합인가
Flutter 선택 이유
단일 코드베이스로 iOS + Android — 6개국 출시에 압도적 효율
렌더링 성능 우수 — 차트/그래프(통계 화면) 부드러움
Hot Reload 매우 빠름 — 개발 속도 극대화
Material 3 + Cupertino 위젯 — 한·일·미 사용자 모두 친숙
Supabase 선택 이유
PostgreSQL 기반 — 복잡한 쿼리(통계, 패턴 분석) 자유로움
실시간 동기화 내장 — 부부 공유 기능에 최적
Row Level Security (RLS) — 보안 자동 적용
오픈소스 — 벤더 락인 적음, 향후 자체 호스팅 가능
무료 시작 — 50K MAU까지 무료, 운영비 절감
글로벌 인프라 — 한·일·미 모두 데이터 센터 있음
8.3 가장 중요한 결정: 오프라인 전략
Supabase는 Firebase처럼 자동 오프라인 캐시가 없음. 육아 앱은 새벽에 인터넷 없이도 기록 가능해야 하므로 오프라인 전략이 절대적으로 중요. 해결책: Brick offline_first 패키지 (사실상 표준).
Brick의 동작
로컬 SQLite에 먼저 저장
인터넷 있으면 백그라운드에서 Supabase에 동기화
충돌 시 last-write-wins 또는 커스텀 로직
다른 기기에서 변경 시 실시간 동기화
